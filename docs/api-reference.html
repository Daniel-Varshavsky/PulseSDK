API Reference

Every public SDK function, and every server endpoint.

SDK — public functions

Everything below lives on the PulseSDK object (com.pulsesdk.PulseSDK) and can be called from anywhere in your app after init().

fun init(context: Context, apiKey: String, serverUrl: String? = null)

Initializes the SDK. Called automatically on app start if the API key is declared in AndroidManifest.xml; call manually otherwise. A no-op if already initialized.

fun identify(externalUserId: String)

Links the SDK's user to your own user identity. Call after login.

fun clearUser()

Unlinks the current user identity. Call on logout.

fun refreshFcmToken(token: String)

Updates the stored push notification token. Call from your FirebaseMessagingService.onNewToken().

fun setTestFcmToken(token: String)

Sets a fake FCM token for development on emulators where real FCM tokens aren't available. Development only.

fun getVariant(experimentName: String): VariantResult?

Returns the variant this user is assigned to for the named experiment. Deterministic and cached — the same user always gets the same result. Returns null if the experiment doesn't exist or isn't active.

fun getActiveVariants(): List<VariantResult>

Returns the assigned variant for every currently active experiment, in one call — useful for a screen that shouldn't need to hardcode experiment names.

fun getExperimentVariants(experimentName: String): List<ExperimentVariantSummary>?

Lists every variant of an experiment, regardless of assignment. Mainly for building QA tooling (e.g. a "preview other variants" switcher) on top of overrideVariant.

fun overrideVariant(experimentName: String, variantId: String)

Forces the local variant assignment to a specific variant for testing. Client-side only — doesn't affect real assignment. Not for production code paths.

fun clearVariantOverride(experimentName: String)

Clears an overrideVariant call.

fun submitStarRating(variantId: String, rating: Int, comment: String? = null, screenId: String? = null)

Submits a 1–5 star rating for a variant.

fun submitThumbs(variantId: String, positive: Boolean, comment: String? = null, screenId: String? = null)

Submits a thumbs up/down response for a variant.

fun submitMultipleChoice(variantId: String, index: Int, comment: String? = null, screenId: String? = null)

Submits a multiple-choice response. index is 0-based into the variant's choices.

fun setCooldownDays(days: Int)

Overrides the default 7-day cooldown used by shouldPrompt.

fun shouldPrompt(screenId: String): Boolean

Whether it's been at least the cooldown period since markPrompted was last called for this screenId. Use this to gate any in-app prompt, not just experiment feedback, without hand-rolling a SharedPreferences timestamp check per screen.

fun markPrompted(screenId: String)

Records that a prompt was just shown for screenId, starting the cooldown window shouldPrompt checks against.

fun submitText(text: String, variantId: String? = null, screenId: String? = null)

Submits open-ended text feedback. Pass variantId = null for general feedback not tied to any experiment.

SDK — internal functions

Not part of the public API (package-private or private in PulseSDK.kt) — listed here only so the SDK's internals aren't a black box.

FunctionPurpose
registerDevice()Registers the device's FCM token on init, before any user identity is known.
fetchActiveExperiments()Fetches and caches the active experiment list once per process.
assignVariant(userId, experiment)The deterministic hash-based assignment used by getVariant/getActiveVariants.
enqueue(...)Writes a feedback event to the local Room queue; backs every submit* call.
scheduleUpload()Enqueues the WorkManager job that flushes the local feedback/crash queues once a network is available.
maybeLogExposure(userId, variantId)Fire-and-forget exposure log, deduped per (user, variant) per process — called from getVariant/getActiveVariants.
installCrashHandler()Chains onto the existing Thread.UncaughtExceptionHandler at init time.
recordCrashBlocking(throwable)Synchronously writes a crash report to the Room queue — runs on the crashing thread, can't wait on a dispatcher hop.
checkInitialized()Guard called at the top of every public function — throws if init() hasn't run.

Endpoints — SDK-facing (API key auth)

Authenticated with an app's API key in the x-api-key header. These are the endpoints the Android SDK itself calls.

EndpointPurpose
POST/devices/registerRegister a device's FCM token, get back a userId
POST/devices/identifyLink a device to an externalUserId
POST/devices/clearUnlink a device from its user
POST/devices/refresh-tokenUpdate a device's FCM token
GET/experimentsList active experiments with their variants (no results — see /experiments/:id/results or /aggregate for those). Accepts appVersion to filter out version-targeted experiments the device doesn't qualify for
POST/feedbackSubmit a feedback response
POST/exposureLog that a user was shown a variant
POST/crashesSubmit a write-ahead-logged crash report

Endpoints — Portal-facing

A mix of JWT (Bearer token, account-scoped) and API-key auth — the Portal uses whichever an action needs.

EndpointAuthPurpose

POST/auth/login—Log in, get a JWT

POST/appsJWTCreate an app
GET/apps/myJWTList apps you're a member of
GET/apps/:idJWTApp details
POST/apps/:id/regenerate-keyJWTRotate an app's API key
POST/apps/:id/membersJWTInvite a collaborator by email
DELETE/apps/:id/members/:memberIdJWTRemove a collaborator
GET/apps/statsAPI keyUsage stats for the current app
POST/experimentsJWTCreate an experiment
PATCH/experiments/:idJWTUpdate an experiment's status, or (while paused) its name/variant weights/choices/metadata/minAppVersion
GET/experiments/:id/resultsAPI keyResults + creator info for one experiment, including exposure counts and response rate
GET/experiments/:id/aggregateAPI keyFull per-variant breakdown (star/thumbs/choice/exposures)
GET/feedbackAPI keyPaginated, filterable raw feedback responses
POST/notifications/sendAPI keySend a push notification to an app's users
